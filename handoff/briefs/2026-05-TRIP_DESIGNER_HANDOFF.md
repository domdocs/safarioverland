## 2. What it does

Staff-facing itinerary builder inside `/admin`. A curator composes a
multi-destination trip (metadata + N chapters + N−1 transits + photos +
practicals), previews it as an editorial document, downloads a
print-ready A4 PDF, and publishes a private public URL to share with
the customer.

### Routes

| Route | Purpose | Auth |
|---|---|---|
| `/admin/itineraries` | List view | admin Basic Auth |
| `/admin/itineraries/new` | Create (title only) | admin |
| `/admin/itineraries/[id]/edit` | Full editor (autosave) | admin |
| `/admin/itineraries/[id]/preview` | Editorial render | admin |
| `/trips/[slug]` | Public read-only (from snapshot) | **none** |
| `/api/admin/itineraries/...` | CRUD + chapters + transits | admin |
| `/api/admin/itineraries/[id]/pdf` | Puppeteer PDF | admin |
| `/api/admin/itineraries/[id]/publish` | Publish/unpublish/dry-run | admin |
| `/api/admin/itineraries/[id]/duplicate` | Clone to draft | admin |

### Data model

- `itineraries` — master row (metadata, theme cols, practicals JSONB)
- `itinerary_chapters` — one per destination (lodge + rhythm JSONB)
- `itinerary_transits` — between adjacent chapters; invariant
  `transits.length === max(0, chapters.length − 1)` enforced in the
  CRUD layer on add/delete/reorder
- `itinerary_reference_sequences` — per-year atomic `SO-YYYY-NNNN` minting
- `itinerary_snapshots` (Phase 5) — immutable publish-time copies;
  `/trips/[slug]` reads the `is_current` snapshot, never live tables

Lib lives in `lib/itineraries/` (types, validate, index/CRUD,
reference, defaults, snapshots, preflight) and `lib/pdf/generate.ts`.

## 3. Non-obvious decisions (read before changing print/PDF code)

This is the part that will save the next person days.

**The PDF is generated by headless Chromium, not Mac Print > PDF.**
Phase 4 was originally deferred to "Niels uses ⌘P". We built it early
because Safari's print pipeline has unfixable-from-CSS quirks:
- `vh` resolves against the **screen** viewport in Safari print, not
  the page area (~211mm not 297mm). Use **`mm` units** for any
  print-page-sized element. There are comments in
  `preview-styles.css` saying exactly this — heed them.
- Safari renders `rgba()` gradients as **opaque dark blocks** in
  print. Chromium renders them correctly. The chapter-hero `::after`
  overlay is back on because we're on Chromium now; don't reintroduce
  the `text-shadow` workaround (it prints as solid rectangles).
- `break-after: page` fires inconsistently in Safari when the element
  doesn't fill the page. Chromium honours it.

**The preview route escapes the admin layout via a route group.**
`app/admin/(chrome)/layout.tsx` holds the sidebar/header chrome. Every
admin route lives under `(chrome)/` **except**
`app/admin/itineraries/[id]/preview/` which sits outside it so the
editorial document is full-bleed. Route groups don't change URLs.
If you add an admin page, put it under `(chrome)/`.

**Puppeteer needs cookies forwarded.** Vercel Deployment Protection
gates preview deployments behind Vercel SSO, separate from our admin
Basic Auth. The PDF route forwards the caller's `Cookie` header
(incl. `_vercel_jwt`) into `page.setCookie()` so Puppeteer reaches
the preview instead of the Vercel login screen. On a custom
production domain without Deployment Protection this is a no-op and
still works. If PDFs ever come back as a Vercel login page, this is
why — check `lib/pdf/generate.ts` `parseCookieHeader`.

**Serverless Chromium.** `@sparticuz/chromium` + `puppeteer-core` in
Vercel functions; full `puppeteer` (devDependency) locally. Split in
`lib/pdf/generate.ts` keyed on `process.env.VERCEL`. All three are in
`serverExternalPackages` in `next.config.mjs` — don't let webpack
bundle them. Cold start ~3–5 s, warm ~1–2 s; `maxDuration = 60`.

**Snapshots are the public source of truth.** Editing a published
itinerary does NOT change `/trips/[slug]` until you re-publish.
Re-publishing keeps the same slug (stable customer URL) and flips the
previous `is_current` snapshot to false. Unpublish only changes
status — the snapshot/URL keep working until manually revoked in
Supabase Studio (set `is_current = false`).

## 4. Theme is locked-by-convention, not by code

`palette` / `typography` / `density` columns + the Phase 5 picker UI
exist and work (savanna/forest/coast, editorial/modern/classic,
spacious/compact). The design is calibrated to
**savanna / editorial / spacious**; the picker has "(default)"
labels nudging curators to leave it. The forest/coast palettes and
modern/classic type pairings were ported from the prototype but have
not been visually QA'd at PDF fidelity — if a curator switches them,
expect rough edges.

## 5. How to operate it

**Create → fill → preview → PDF → publish:**
1. `/admin/itineraries` → "+ New itinerary" → title → redirected to edit
2. Fill metadata, add chapters (place + coords are required for the
   maps), upload photos, edit auto-generated transits
3. "Preview ↗" opens the editorial render in a new tab
4. "Download PDF" streams a Chromium-rendered A4 PDF
   (`SO-YYYY-NNNN-LASTNAME-YYYY-MM.pdf`)
5. "Publish" runs preflight; fix any inline-listed issues; on success
   a public `/trips/[slug]` URL is minted — "Copy public URL" button
   appears in the header
6. "Duplicate" clones a trip into a fresh draft for next season

**Pre-flight rules** (`lib/itineraries/preflight.ts`) reject publish
if: no cover photo, zero chapters, a chapter missing coords/dates/
epigraph/intro/hero, a transit still showing placeholder values
(`Transfer`/`TBD`/`To be specified.`), or transit count out of sync.

  path (OS denies executing `node_modules/.bin/next`); all UI
  verification was done on Vercel preview deploys. If you can run
  the dev server elsewhere, prefer that.

## 7. Deferred: templates library

The one remaining item from the original "out of scope" list worth
doing. Scope sketch for whoever picks it up:

- New table `itinerary_templates` (or reuse snapshots with a
  `kind = 'template'` flag) — a saved chapter or whole-trip shape,
  detached from any one itinerary.
- "Save as template" action on a chapter card and on the trip.
- "Insert template" picker in the editor that deep-copies the
  template's content into the current trip (mind the
  chapter→transit invariant — inserting a chapter must regenerate
  the transit chain, reuse `createChapter`'s placeholder logic).
- Photos: templates should probably store no photo URLs (they're
  trip-specific) — blank them on insert.

Recommend writing a proper brief for it after the team has run the
publish flow on a few real trips and knows what they actually reach
for repeatedly. Don't speculatively build the library before that.

## 8. Known rough edges (not blockers)

- Sparse chapters (metadata but no intro/lodge/rhythm) render a
  thin page in the PDF. Preflight now blocks publishing these, but
  the draft preview will still look sparse — that's expected.
- First "Download PDF" after a deploy is slow (Chromium cold start).
  Retry is fast. Consider a Vercel cron ping to keep the function
  warm if volume grows.
- `handoff/briefs/2026-05-SUPABASE_SECURITY_HARDENING.md` rode along
  in an early Phase 1–4 commit via a stray `git add -A` — it's
  unrelated to Trip Designer; it's a separate advisor brief, leave
  or relocate as you see fit.
